# Upgrade Script — projects 3.x → 4.0.0

**MAJOR-version migration (required reading before completing the upgrade).** The 4.0 model:
projects are two-tier (org-public commons by default; private in the owner's My Drive,
invisible-until-shared); ideas are private-and-invisible by default in owners' My Drives with
share/promote/unshare flows; artifacts inherit their parent's access structurally; discovery via
the pointer index `/shared/projects-index/` (the 3.x `projects-manifest.json` is retired).

## ⚠ THE ORDERING RULE (review before provisioning)

upgrade-collection Step 6.5 will flag provisioning (new `collaborative-acls.json` + setup-interview
changes). **Run this script's admin half BEFORE `@ai:install-collection projects` applies the
`/shared/projects/` all@ WRITER grant.** Two reasons: (1) the gated-idea review below — any 3.x
idea folder carrying narrower-than-baseline ACLs would have its gating semantics changed by new
uniform grants; (2) the pointer index must exist and be populated so listings don't go dark
between file-sync and provisioning.

NOTE: on many 3.x installs (including this org) an all@ writer grant on `/shared/projects/`
already exists operationally — Step 5.5 will then verify/no-op. The review is still mandatory:
verify ACL state live, never assume (checklist rule).

## Admin half (run before provisioning)

1. **Project inventory:** `aifs_list("/shared/projects/")`. A directory is a PROJECT iff it
   contains `project.md` (or a 3.x `project.md`-equivalent brief). **Directories without it
   (e.g. org record spaces like `core-improvements/`) are NOT projects — leave them completely
   untouched.** For each project: read `project.md` → build the 4.0 pointer at
   `/shared/projects-index/{owner_hash}-{slug}.json` with `scope: "org_public"`,
   `location: {"path": ...}`, `status` carried over, owner from the 3.x record (`created_by` /
   owner field; fall back to the admin with a `"migration": "owner-unknown"` marker).
2. **Gated-idea review** (client-intelligence precedent): for each project's `ideas/` residents,
   `aifs_get_permissions` and compare **roles per subject against the project baseline** (the
   dunder-mifflin lesson: role divergence is the tell, `inherited_from` misreports). Divergent
   (gated) ideas → flag; resolve with the idea's author: relocate private (the author runs the
   new create-idea/share-idea flow to pull it into their space; admin then stubs the resident
   copy) or open to project tier (clear the divergent ACLs via helper for hygiene). Expectation
   for most 3.x orgs: ZERO gated ideas (3.x per-idea ACLs were planned, not shipped) — the review
   should be a fast verification pass.
3. **Members' local private ideas** (3.x stored private ideas in members' LOCAL workspaces):
   nothing to migrate centrally — each member's ideas remain local-and-invisible; share-idea
   offers the one-time move into their My Drive on first touch (member half).
4. **Retire `projects-manifest.json`:** overwrite with `{"retired": "4.0.0", "see":
   "/shared/projects-index/"}`.
5. **Provision:** `@ai:install-collection projects` — upgrade interview records
   `projects_default_visibility` (org_public_first) + `ideas_default_visibility` (private_first),
   retires `shared_projects_path` and `ideas_require_private_stage`; Setup Completion creates
   `/shared/projects-index/`; Step 5.5 applies/verifies the grants.

## Member half (surfaced during members' @ai:update of the 4.0 entry)

- Nothing mandatory. New verbs: tier prompts at creation, share-idea's three moves,
  transition-project. Private 3.x local ideas get a one-time move offer on first touch.
- Members flagged in the gated-idea review (if any) complete their relocations.

## Verification (admin, after both halves)

- Every real project has a 4.0 pointer; non-project directories untouched; manifest retired.
- `aifs_get_permissions("/shared/projects/")` and `/shared/projects-index/` show the all@ writer
  grants (live read, not narration).
- A non-admin can: create an org-public project, create a private project (invisible — admin
  gets PATH_NOT_FOUND on its folder), write an action item with attribution (test plan 12,
  Phases U/PJ).
